    Documentation is available on each github wiki on each library

    As there is no pull request mechanism for wiki repositories, we need to do it manually

    Here is the process to modify the documentation.

    • Fork the library. for example: codjo-release-test
    • Create the wiki on the newly forked repository and create a new dummy page to instanciate the codjo-release-test.wiki repository
    • Clone the wiki repository on your local workstation. 
      git clone https://github.com/codjo-sandbox/codjo-release-test.wiki.git
    • Using command line, go in the new directory and add a remote on codjo's wiki 
      git remote add codjo https://github.com/codjo/codjo-release-test.wiki.git
    • Using command line, update the remote branch codjo 
      git pull codjo master
    • Do your modification on the documentation, using the github GUI or on your local files
    • Test your modifications, commit and push your modifications on the forked repository
      git push origin
    • Check your modifications on the forked wiki
    • If everything is OK, update from origin 
      git pull origin
      then update the remote branch codjo, change your connection in your netrc file, commit and push your modifications
      git push codjo master
